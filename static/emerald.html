First-Time Setup (do this once)
1

Get the Emerald Legacy source code

Open Terminal and run:

git clone https://github.com/cRz-Shadows/Pokemon_Emerald_Legacy

This creates a Pokemon_Emerald_Legacy folder — that’s your Source Directory. Never modify files inside it; the randomizer reads from it and writes to a separate output folder.

2

Install devkitARM via devkitPro

Follow the instructions at devkitpro.org/wiki/Getting_Started to install devkitPro and select the GBA development package. This installs the arm-none-eabi-gcc toolchain used by Emerald Legacy’s Makefile.

⚠️ Make sure $DEVKITARM is set in your environment. The Emerald Legacy Makefile requires it.
3

Create an Output Directory

Make a new empty folder anywhere on your Mac — for example ~/Desktop/ELRando_Output. This is where the randomized source files are written. It is fully overwritten on every run, so don’t store anything important here.

Generating a Randomized ROM (every run)
1

Set your directories

At the top of the app, click Browse… next to Source Directory and select your cloned Pokemon_Emerald_Legacy folder.

Click Browse… next to Output Directory and select the empty output folder you created.

2

Set a seed

A random seed is generated automatically each time you open the app. You can also type any number in the Seed field on the Randomize tab, or click 🎲 to roll a new one. The seed fully determines the randomization — the same seed with the same settings always produces the same ROM.

3

Configure your options

Use the tabs to customize each category. Every tab is independent — you can randomize starters but leave trainers unchanged, for example.

4

Randomize & Build ROM

Make sure Build ROM after randomizing is checked (it is by default), then click the Randomize & Build ROM button.

The app will: copy the source → apply all changes → compile with make → open a save dialog so you can name and place the .gba file anywhere.

ℹ️ If you uncheck “Build ROM”, the randomized source is saved to the Output Directory but not compiled. You can compile it yourself later by opening Terminal in that folder and running make.
5

Play the ROM

Load the saved .gba file in any GBA emulator — mGBA is the recommended choice. The file is a standard GBA ROM and works on real hardware via a flash cart as well.

Tab Reference
⚙️
General

Limit Pokémon — Restricts the global pool to Gen 1 only (Bulbasaur–Mew), Gen 3 only (Treecko–Deoxys), or all 386. This filter applies everywhere: starters, wild encounters, trainers, and static Pokémon all draw from the same filtered pool.

Make Evolutions Easier — Caps evolution levels so middle-stage Pokémon evolve by Lv 30 and final-stage by Lv 40.

Full HM Compatibility — Gives every Pokémon the ability to learn all HM moves. Strongly recommended for randomized playthroughs.

🌱
Starters

Unchanged — Keeps Treecko, Torchic, and Mudkip.

Custom — You pick exactly which 3 Pokémon fill the starter slots using searchable dropdowns.

Random — 3 distinct Pokémon chosen at random from the pool.

Random (basic with 2 evolutions) — Only picks from Pokémon that are the first stage of a 3-stage line, giving a classic “starter feel.”

Randomize Starter Held Item — Replaces the held item each starter comes with. Enable Ban Bad Items to ensure starters get useful items.

🌿
Wild Pokémon

Random — Every wild slot is independently replaced. Area 1-to-1 — consistent mapping per area. Global 1-to-1 — one mapping used across the whole game.

Additional Rules (available when not Unchanged): Similar Strength picks replacements with a similar BST. Catch ’Em All ensures every Pokémon appears at least once. Type Themed Areas assigns each route a random type.

Set Minimum Catch Rate — makes Pokémon easier to catch. At max (5), any ball is a guaranteed catch.

Levels are never changed — only species are replaced.

🧢
Trainers

Random — Each slot is replaced with a random Pokémon. Random (Even) — the pool is cycled so every species appears about equally often. Type Themed — every trainer gets a random type theme. Type Themed (Bosses Only) — only gym leaders and Elite Four are type-themed.

Force Fully Evolved at Level X — any trainer Pokémon at or above the chosen level that isn’t its final evolution is upgraded after randomization.

Levels, moves, and held items are always preserved — only species are replaced.

🗿
Static Pokémon

Static Pokémon are encountered at fixed overworld locations — not in grass or trainer battles. Examples: the Regi trio, Groudon, Kyogre, Rayquaza, Latias/Latios, and gift Pokémon.

Shuffle — Statics swap among their own locations. Random — anything can replace anything. Random (Similar Strength) — replacements have a similar Base Stat Total.

🤝
In-Game Trades

NPC trades built into the game. Randomize Given Only — what you receive is randomized; what you must give stays the same. Randomize Both — both sides of the trade are randomized.

Sub-options let you also randomize the received Pokémon’s nickname, OT name, IVs, and held item independently.

🎒
Items

Controls overworld field items. Key items and HMs are never replaced.

Shuffle — same items redistributed across locations. Random — each location independently gets a random item. Random (Even) — all items appear as equally often as possible.

Ban Bad Items — excludes cheap/situational items so every replaced item is useful.

Starting Items lets you pre-load your bag and PC with specific items at the start of a new game.

Seeds & Sharing
✅ The seed fully determines the randomization. The same seed number with the same settings always produces an identical ROM — great for racing or group playthroughs.
  • Write down or share your Seed number before clicking Randomize.
  • Everyone using the same seed must also use the same settings on every tab for results to match.
  • Click 🎲 to roll a completely random new seed at any time.
  • You can type any integer as a seed — short numbers like 12345 work fine.
  • The seed is also printed at the top of the build log so you always have a record.
Tips & Notes
  • The Source Directory is never modified. The randomizer copies the entire source to the Output Directory first, then applies all changes there. Your original Emerald Legacy source stays clean.
  • The Output Directory is overwritten completely on every run. Don’t store anything important in it — only keep your saved .gba files, which you place elsewhere via the save dialog.
  • Recommended settings for a fun randomized run: Wild = Random, Trainers = Random, Full HM Compatibility = ON. These give the best experience without making the game impossible to complete.
  • Make sure devkitARM is properly installed and $DEVKITARM is set in your shell environment. If the build fails with a compiler error, this is the most likely cause.
  • The log panel shows exactly what changed. Click ▶ Show Log at the bottom of the window to see every species replacement, item change, and compiler line.
  • The .gba file works in any GBA-compatible emulator (mGBA, RetroArch) and on real hardware via an EZ-Flash or similar flash cartridge.
