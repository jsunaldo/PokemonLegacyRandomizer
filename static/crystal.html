First-Time Setup (do this once)
1

Get the Crystal Legacy source code

Open Terminal and run:

git clone https://github.com/cRz-Shadows/Pokemon_Crystal_Legacy

This creates a Pokemon_Crystal_Legacy folder — that's your Source Directory. Never modify files inside it; the randomizer reads from it and writes to a separate output folder.

2

Install RGBDS v0.5.2

Download the macOS binary from github.com/gbdev/rgbds/releases/tag/v0.5.2 (file: rgbds-0.5.2-macos-x86-64.zip), unzip it, then in Terminal:

sudo cp rgbasm rgblink rgbfix rgbgfx /usr/local/bin/

⚠️ Do not install the latest RGBDS. Crystal Legacy compiles only with v0.5.2. If you already have a newer version you'll need to replace it with 0.5.2.
3

Create an Output Directory

Make a new empty folder anywhere on your Mac — for example ~/Desktop/CLRando_Output. This is where the randomized source files are written. It is fully overwritten on every run, so don't store anything important here.

Generating a Randomized ROM (every run)
1

Set your directories

At the top of the app, click Browse next to Source Directory and select your cloned Pokemon_Crystal_Legacy folder.

Click Browse next to Output Directory and select the empty output folder you created.

2

Set a seed

A random seed is generated automatically each time you open the app. You can also type any number in the Seed field, or click 🎲 to roll a new one. The seed fully determines the randomization — the same seed with the same settings always produces the same ROM.

3

Configure your options

Use the tabs to customize each category. Every tab is independent — you can randomize starters but leave trainers unchanged, for example. See the Tab Reference section below for a full breakdown of every option.

4

Randomize & Build ROM

Make sure Build ROM after randomizing is checked (it is by default), then click the Randomize & Build ROM button.

The app will: copy the source → apply all changes → compile with make → open a save dialog so you can name and place the .gbc file anywhere you like.

ℹ️ If you uncheck "Build ROM", the randomized ASM source is saved to the Output Directory but not compiled. You can compile it yourself later by opening a Terminal in that folder and running make.
5

Play the ROM

Load the saved .gbc file in any Game Boy Color emulator — mGBA and BGB are popular choices. The file is a standard GBC ROM and works on real hardware via a flash cart as well.

Tab Reference
⚙️
General

Limit Pokémon — Restricts the global pool to Gen 1 only (Bulbasaur–Mew), Gen 2 only (Chikorita–Celebi), or all 251. This filter applies everywhere: starters, wild encounters, trainers, and static Pokémon all draw from the same filtered pool.

Make Evolutions Easier — Caps evolution levels so middle-stage Pokémon evolve by Lv 30 and final-stage by Lv 40, making trade evolutions like Haunter and Kadabra reachable without trading.

Remove Time-Based Evolutions — Converts day/night happiness evolutions to item evolutions: Espeon uses a Sun Stone, Umbreon uses a Moon Stone.

Full HM Compatibility — Gives every Pokémon the ability to learn all 7 HMs (Cut, Fly, Surf, Strength, Flash, Whirlpool, Waterfall). Strongly recommended for randomized playthroughs to prevent getting stuck because no party member can learn a required field move.

🌱
Starters

Unchanged — Keeps Cyndaquil, Totodile, and Chikorita.

Custom — You pick exactly which 3 Pokémon fill the starter balls using dropdown menus.

Random (Completely) — 3 distinct Pokémon chosen at random from the pool.

Random (basic with 2 evolutions) — Only picks from Pokémon that are the first stage of a 3-stage line (e.g. Bulbasaur, Charmander), giving a classic "starter feel."

Randomize Starter Held Item — Replaces the held item each starter comes with. Enable Ban Bad Items to ensure starters get useful items like Leftovers or Scope Lens.

When starters are randomized, the Elm's Lab dialogue ("You'll take CYNDAQUIL, the fire Pokémon?") is automatically updated to show the correct species name and type.

🌿
Wild Pokémon

Random — Every wild slot is independently replaced. Area 1-to-1 — consistent mapping per area (same species always maps to the same replacement within one route). Global 1-to-1 — one mapping used across the whole game.

Additional Rules (available when not Unchanged): Similar Strength picks replacements with a similar Base Stat Total. Catch 'Em All ensures every Pokémon in the pool appears at least once in the wild. Type Themed Areas assigns each route a random type — all encounters there share that type.

Use Time-Based Encounters — randomizes morning/day/night slots independently. Uncheck if playing on an emulator without a clock.

Randomize Held Items — randomizes the items wild Pokémon may hold when encountered.

Levels are never changed — only species are replaced.

🧢
Trainers

Random — Each trainer slot is replaced with a random Pokémon. Random (Even Distribution) — the pool is cycled so every species appears about equally often. Type Themed — every trainer gets a random type theme. Type Themed (Bosses Only) — only gym leaders, Elite Four, and the Rival are type-themed; regular trainers are randomly randomized.

Rival Carries Starter — the Rival always has the starter that counters yours (and its evolutions), just like the original game. Works with randomized starters.

Similar Strength — replacements match the original Pokémon's Base Stat Total, keeping the difficulty curve intact.

Force Fully Evolved at Level X — any trainer Pokémon at or above the chosen level that isn't its final evolution is upgraded to its final form after randomization.

Levels, moves, held items, and DVs are always preserved — only species are replaced.

🗿
Static Pokémon

Static Pokémon are encountered at fixed overworld locations — not in grass or trainer battles. Examples: Red Gyarados, Sudowoodo, Lugia, Ho-Oh, Suicune, roaming Raikou & Entei, Snorlax, Lapras, and gift Pokémon (Eevee, Dratini, fossils).

Swap — Legendaries swap with legendaries; standard statics swap with non-legendaries. Random (Completely) — anything can replace anything. Random (Similar Strength) — replacements have a similar Base Stat Total.

🤝
In-Game Trades

NPC trades built into the game — for example giving a Bellsprout and receiving an Onix.

Randomize Given Only — what you receive is randomized; what you must give stays the same. Randomize Both — both sides of the trade are randomized.

Sub-options let you also randomize the received Pokémon's nickname, OT name, IVs (DVs), and held item independently.

🎒
Items

Controls overworld items — both visible Pokéball sprites and hidden items found with the Itemfinder. Key items and HMs are never replaced.

Shuffle — the same set of items is redistributed across locations (every item still exists, just in a different spot). Random — each location independently gets a random item. Random (Evenly Distribute) — all items in the pool appear as close to equally often as possible.

Ban Bad Items — excludes cheap/situational items (basic Potions, Antidotes, plain Repel) so every replaced item is at least somewhat useful. Applies to Random and Even modes only.

Seeds & Sharing
✅ The seed fully determines the randomization. The same seed number with the same settings always produces an identical ROM — great for racing or group playthroughs.
  • Write down or share your Seed number (shown on the Randomize tab) before clicking Randomize.
  • Everyone using the same seed must also use the same settings on every tab for results to match.
  • Click 🎲 to roll a completely random new seed at any time.
  • You can type any integer as a seed — short numbers like 12345 work fine.
  • The seed is also printed at the top of the build log so you always have a record.
Tips & Notes
  • The Source Directory is never modified. The randomizer copies the entire source to the Output Directory first, then applies all changes there. Your original Crystal Legacy source stays clean.
  • The Output Directory is overwritten completely on every run. Don't store anything important in it — only keep your saved .gbc files, which you place elsewhere via the save dialog.
  • Recommended settings for a fun randomized run: Wild = Random, Trainers = Random, Full HM Compatibility = ON. These give the best experience without making the game impossible to complete.
  • RGBDS version matters. Only v0.5.2 works. If you see assembler errors mentioning MAJOR or undefined macros, you have a newer RGBDS installed. Replace it with v0.5.2.
  • The log panel shows exactly what changed. Click ▶ Show Log at the bottom of the window to see every species replacement, item change, and compiler line. If the build fails, the error will appear there.
  • Playing without a real-time clock? Most emulators need to be configured with a clock, or use one that supports it (mGBA does). If you play without a clock, uncheck Use Time-Based Encounters so morning/day/night slots all show the same Pokémon.
  • The .gbc file works in any GBC-compatible emulator (mGBA, BGB, RetroArch) and on real hardware via an EZ-Flash or similar flash cartridge.
