Static Pokémon are encountered at fixed overworld locations — not in wild grass or trainer parties. Examples: Snorlax (×2), and gift Pokémon such as Lapras, Eevee, fossils, and Magikarp.

First-Time Setup (do this once)
1

Get the Yellow Legacy source code

Open Terminal and run:

git clone https://github.com/cRz-Shadows/Pokemon_Yellow_Legacy

This creates a Pokemon_Yellow_Legacy folder — that's your Source Directory. Never modify files inside it; the randomizer reads from it and writes to a separate output folder.

2

Install RGBDS 0.6.0+

Download the macOS binary for v0.6.0 or newer from github.com/gbdev/rgbds/releases, unzip it, then in Terminal:

sudo cp rgbasm rgblink rgbfix rgbgfx /usr/local/bin/

⚠️ Yellow Legacy requires RGBDS 0.6.0 or newer. If you also use Crystal Legacy (which needs the older 0.5.2), you'll need to swap the binaries in /usr/local/bin/ when switching between games.
3

Create an Output Directory

Make a new empty folder anywhere on your Mac — for example ~/Desktop/YLRando_Output. This is where the randomized source files are written. It is fully overwritten on every run, so don't store anything important here.

Generating a Randomized ROM (every run)
1

Set your directories

At the top of the app, click Browse next to Source Directory and select your cloned Pokemon_Yellow_Legacy folder.

Click Browse next to Output Directory and select the empty output folder you created.

2

Set a seed

A random seed is generated automatically each time you open the app. You can also type any number in the Seed field, or click 🎲 to roll a new one. The seed fully determines the randomization — the same seed with the same settings always produces the same ROM.

3

Configure your options

Use the tabs to customize each category. Every tab is independent — you can randomize starters but leave trainers unchanged, for example.

4

Randomize & Build ROM

Make sure Build ROM after randomizing is checked (it is by default), then click the Randomize & Build ROM button.

The app will: copy the source → apply all changes → compile with make → open a save dialog so you can name and place the .gb file anywhere you like.

ℹ️ If you uncheck "Build ROM", the randomized ASM source is saved to the Output Directory but not compiled. You can compile it yourself later by opening a Terminal in that folder and running make.
5

Play the ROM

Load the saved .gb file in any Game Boy emulator — mGBA and BGB are popular choices. The file works on real hardware via a flash cart as well.

Tab Reference
⚙️
General

Make Evolutions Easier — Caps evolution levels so middle-stage Pokémon evolve by Lv 30 and final-stage by Lv 40 (e.g. Haunter and Kadabra, normally high-level in Yellow Legacy, evolve at Lv 40).

Full HM Compatibility — Gives every Pokémon the ability to learn all 5 HMs (Cut, Fly, Surf, Strength, Flash). Strongly recommended for randomized playthroughs to prevent getting stuck.

🌱
Starters

Unchanged — Keeps Bulbasaur, Charmander, and Squirtle at their respective gift locations.

Custom — You pick exactly which Pokémon replaces all three gift starter locations.

Random (Completely) — One Pokémon chosen at random from the pool; all three gift locations give the same species.

Random (basic with 2 evolutions) — Only picks from Pokémon that are the first stage of a 3-stage line (e.g. Bulbasaur, Charmander), giving a classic "starter feel."

🌿
Wild Pokémon

Random — Every wild slot is independently replaced. Area 1-to-1 — consistent mapping per area. Global 1-to-1 — one mapping used across the whole game.

Additional Rules: Similar Strength picks replacements with a similar Base Stat Total. Catch 'Em All ensures every Pokémon in the pool appears at least once. Type Themed Areas assigns each route a random type.

Levels are never changed — only species are replaced. Fishing (Old Rod, Good Rod, Super Rod) is also randomized when wild mode is active.

🧢
Trainers

Random — Each trainer slot is replaced with a random Pokémon. Random (Even Distribution) — the pool is cycled so every species appears about equally often. Type Themed — every trainer gets a random type theme. Type Themed (Bosses Only) — only gym leaders and Elite Four are type-themed.

Similar Strength — replacements match the original Pokémon's Base Stat Total.

Force Fully Evolved at Level X — any trainer Pokémon at or above the chosen level that isn't its final evolution is upgraded to its final form after randomization.

Levels and moves are always preserved — only species are replaced.

🗿
Static Pokémon

Static Pokémon are encountered at fixed overworld locations. Examples: Snorlax (Route 12 and Route 16), and gift Pokémon — Lapras (Silph Co.), Eevee (Celadon Mansion), fossil Pokémon, and Magikarp (Mt. Moon Pokémon Center).

The legendary birds (Articuno, Zapdos, Moltres) and Mewtwo use a trainer-battle system and are not randomized.

Swap — Legendaries swap with legendaries; standard statics swap with non-legendaries. Random (Completely) — anything can replace anything. Random (Similar Strength) — replacements have a similar Base Stat Total.

🤝
In-Game Trades

NPC trades built into the game — giving one Pokémon and receiving another.

Randomize Given Only — what you receive is randomized; what you must give stays the same. Randomize Both — both sides of the trade are randomized.

Yellow trade format does not support OT names, DVs, or held items — only species and nickname can be changed.

🎒
Items

Controls overworld items — both visible Pokéball sprites and hidden items found with the Itemfinder. Key items and HMs are never replaced.

Shuffle — the same set of items is redistributed across locations. Random — each location independently gets a random item. Random (Evenly Distribute) — all items in the pool appear as equally often as possible.

Ban Bad Items — excludes cheap/situational items so every replaced item is at least somewhat useful.

Zero Grinding — adds Rare Candy to Viridian Mart for $10, available from the very start of the game.

Elite 4 Prep — stocks the Indigo Plateau Lobby shop with Rare Candy, Full Restore, Max Elixir, and Max Revive for $10 each.

Seeds & Sharing
✅ The seed fully determines the randomization. The same seed number with the same settings always produces an identical ROM — great for racing or group playthroughs.
  • Write down or share your Seed number (shown on the Randomize tab) before clicking Randomize.
  • Everyone using the same seed must also use the same settings on every tab for results to match.
  • Click 🎲 to roll a completely random new seed at any time.
  • You can type any integer as a seed — short numbers like 12345 work fine.
  • The seed is also printed at the top of the build log so you always have a record.
Tips & Notes
  • The Source Directory is never modified. The randomizer copies the entire source to the Output Directory first, then applies all changes there. Your original Yellow Legacy source stays clean.
  • The Output Directory is overwritten completely on every run. Don't store anything important in it — only keep your saved .gb files, which you place elsewhere via the save dialog.
  • Recommended settings for a fun randomized run: Wild = Random, Trainers = Random, Full HM Compatibility = ON. These give the best experience without making the game impossible to complete.
  • The log panel shows exactly what changed. Click ▶ Show Log at the bottom of the window to see every species replacement and compiler line. If the build fails, the error will appear there.
  • The .gb file works in any GB-compatible emulator (mGBA, BGB, RetroArch) and on real hardware via an EZ-Flash or similar flash cartridge.
